## 測試分類

| 測試類型 | 目的 | 工具 |
|----------|------|------|
| 負載測試（Load Test） | 正常峰值下的表現 | k6 / Artillery |
| 壓力測試（Stress Test） | 找出系統極限與崩潰點 | k6 |
| 浸泡測試（Soak Test） | 長時間穩定性（記憶體洩漏） | k6 |
| 峰值測試（Spike Test） | 突然湧入流量的反應 | k6 |
| 單元效能測試 | bcrypt / DB query 個別耗時 | Jest + 計時 |

## 二、負載測試腳本（k6）

### 2-1 基礎登入壓力測試

```js
// tests/load/login-load.js
import http from 'k6/http';
import { check, sleep } from 'k6';
import { Rate } from 'k6/metrics';

const errorRate = new Rate('errors');

export const options = {
  stages: [
    { duration: '1m', target: 1000 },    // 1 分鐘爬升到 1000 VU
    { duration: '3m', target: 10000 },   // 3 分鐘爬升到 1 萬
    { duration: '5m', target: 100000 },  // 5 分鐘爬升到 10 萬
    { duration: '5m', target: 100000 },  // 維持 10 萬 5 分鐘
    { duration: '2m', target: 0 },       // 2 分鐘降回 0
  ],
  thresholds: {
    http_req_duration: ['p(95)<500'],  // 95% 請求在 500ms 內完成
    http_req_failed: ['rate<0.01'],    // 失敗率 < 1%
    errors: ['rate<0.05'],
  },
};

export default function () {
  const payload = JSON.stringify({
    account: `user_${__VU}@test.com`,  // 每個 VU 用不同帳號
    password: 'Test1234!',
  });

  const params = {
    headers: { 'Content-Type': 'application/json' },
    timeout: '10s',
  };

  const res = http.post('http://localhost:3000/api/auth/login', payload, params);

  const ok = check(res, {
    'status is 200': (r) => r.status === 200,
    'has token': (r) => r.json('data.token') !== undefined,
    'response time < 500ms': (r) => r.timings.duration < 500,
  });

  errorRate.add(!ok);
  sleep(1);
}
```

### 2-2 行事曆 CRUD 綜合測試

```js
// tests/load/schedule-crud.js
import http from 'k6/http';
import { check, sleep, group } from 'k6';

export const options = {
  vus: 10000,
  duration: '10m',
  thresholds: {
    'http_req_duration{type:create}': ['p(95)<800'],
    'http_req_duration{type:query}': ['p(95)<300'],
    'http_req_duration{type:update}': ['p(95)<500'],
    http_req_failed: ['rate<0.02'],
  },
};

// 先登入取得 token（setup 階段）
export function setup() {
  const res = http.post('http://localhost:3000/api/auth/login', JSON.stringify({
    account: 'loadtest@test.com',
    password: 'Test1234!',
  }), { headers: { 'Content-Type': 'application/json' } });
  return { token: res.json('data.token') };
}

export default function (data) {
  const headers = {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${data.token}`,
  };

  group('建立行程', function () {
    const now = new Date();
    const start = new Date(now.getTime() + __VU * 3600000).toISOString();
    const end = new Date(now.getTime() + __VU * 3600000 + 1800000).toISOString();

    const res = http.post('http://localhost:3000/api/schedule', JSON.stringify({
      title: `測試活動 ${__VU}-${__ITER}`,
      start_time: start,
      end_time: end,
      is_public: false,
    }), { headers, tags: { type: 'create' } });

    check(res, { '建立成功': (r) => r.status === 201 });
  });

  group('查詢行程', function () {
    const res = http.post('http://localhost:3000/api/schedule/query', JSON.stringify({
      start_time: '2026-01-01T00:00:00Z',
      end_time: '2026-12-31T23:59:59Z',
    }), { headers, tags: { type: 'query' } });

    check(res, { '查詢成功': (r) => r.status === 200 });
  });

  sleep(Math.random() * 2 + 0.5); // 0.5~2.5 秒隨機 think time
}
```

### 2-3 峰值測試（突發流量）

```js
// tests/load/spike-test.js
import http from 'k6/http';
import { check } from 'k6';

export const options = {
  stages: [
    { duration: '30s', target: 100 },    // 正常流量
    { duration: '10s', target: 50000 },  // 突然暴增（促銷/事件觸發）
    { duration: '1m', target: 50000 },   // 維持峰值
    { duration: '10s', target: 100 },    // 回落
    { duration: '1m', target: 100 },     // 觀察恢復
  ],
  thresholds: {
    http_req_duration: ['p(99)<2000'],   // 峰值允許放寬到 2 秒
    http_req_failed: ['rate<0.05'],      // 峰值失敗率允許 5%
  },
};

export default function () {
  const res = http.get('http://localhost:3000/api/user/info', {
    headers: { Authorization: 'Bearer <valid_token>' },
  });
  check(res, { 'ok': (r) => r.status === 200 || r.status === 429 });
}
```

### 2-4 浸泡測試（記憶體洩漏檢測）

```js
// tests/load/soak-test.js
import http from 'k6/http';
import { check, sleep } from 'k6';

export const options = {
  vus: 1000,
  duration: '2h',  // 持續 2 小時
  thresholds: {
    http_req_duration: ['p(95)<500'],
    http_req_failed: ['rate<0.01'],
  },
};

export default function () {
  // 混合各種請求
  const rand = Math.random();
  if (rand < 0.4) {
    http.get('http://localhost:3000/api/user/info');
  } else if (rand < 0.7) {
    http.post('http://localhost:3000/api/schedule/query', JSON.stringify({
      start_time: '2026-01-01T00:00:00Z',
      end_time: '2026-12-31T23:59:59Z',
    }));
  } else {
    http.post('http://localhost:3000/api/auth/login', JSON.stringify({
      account: 'soak@test.com',
      password: 'Test1234!',
    }));
  }
  sleep(1);
}
```

## 四、測試前置準備

### 4-1 建立測試用戶腳本

```js
// tests/load/setup-test-users.js
// 執行: node tests/load/setup-test-users.js
const axios = require('axios');

const BASE_URL = 'http://localhost:3000';
const USER_COUNT = 100000;

async function createUsers(count) {
  const batchSize = 100;
  for (let i = 0; i < count; i += batchSize) {
    const promises = [];
    for (let j = i; j < Math.min(i + batchSize, count); j++) {
      promises.push(
        axios.post(`${BASE_URL}/api/auth/register`, {
          email: `user${j}@loadtest.com`,
          username: `user${j}`,
          account: `user${j}`,
          password: 'Test1234!',
          passwordChk: 'Test1234!',
        }).catch(() => {}) // 忽略已存在的帳號
      );
    }
    await Promise.all(promises);
    console.log(`建立用戶進度: ${Math.min(i + batchSize, count)} / ${count}`);
  }
}

createUsers(USER_COUNT).then(() => console.log('完成'));
```

## 五、監控指標與觀測

### 5-1 k6 關鍵指標解讀

| 指標 | 說明 | 目標值 |
|------|------|--------|
| `http_req_duration p(95)` | 95% 請求完成時間 | < 500ms |
| `http_req_duration p(99)` | 99% 請求完成時間 | < 1000ms |
| `http_req_failed` | 失敗率 | < 1% |
| `http_reqs` | 每秒請求數（RPS） | 觀察最大值 |
| `vus` | 當前虛擬用戶數 | 確認達到目標 |

### 5-2 伺服器端監控指令

```bash
# CPU / 記憶體即時監控
top -p $(pgrep -d',' node)

# PostgreSQL 連線數監控
psql -c "SELECT count(*) FROM pg_stat_activity WHERE state = 'active';"

# 每 5 秒監控一次 DB 連線
watch -n 5 "psql -c \"SELECT state, count(*) FROM pg_stat_activity GROUP BY state;\""

# Node.js 記憶體用量（PM2）
pm2 show backend

# 日誌錯誤計數
tail -f logs/api.log | grep -c "ERROR"
```

### 5-3 慢查詢分析

```sql
-- 開啟慢查詢日誌（postgresql.conf）
-- log_min_duration_statement = 100   -- 超過 100ms 的 query 都記錄

-- 查看目前執行中的慢查詢
SELECT pid, now() - query_start AS duration, query, state
FROM pg_stat_activity
WHERE state != 'idle' AND now() - query_start > interval '1 second'
ORDER BY duration DESC;

-- 查詢最慢的 SQL（需開啟 pg_stat_statements extension）
SELECT query, calls, mean_exec_time, max_exec_time
FROM pg_stat_statements
ORDER BY mean_exec_time DESC
LIMIT 10;
```

## 九、常見問題與排查

| 現象 | 可能原因 | 排查方式 |
|------|----------|----------|
| 大量 ECONNREFUSED | Port 未開放 / 服務崩潰 | `pm2 logs`、`netstat -an | grep 3000` |
| 大量 429 | Rate limit 觸發 | 確認 `/api` 路由限流設定 |
| 大量 403 | IP 白名單阻擋 | 確認 `ipWhitelist.js` 設定 |
| DB 連線超時 | Pool 耗盡 | `pg_stat_activity` 查看連線數 |
| Node.js 記憶體暴增 | 記憶體洩漏 | `pm2 show backend` 觀察 heap |
| bcrypt 延遲高 | Thread pool 不足 | 提高 `UV_THREADPOOL_SIZE` |
